# Internationalisation

## Locales
Hardcoded in two places (must match):
- `src/i18n/routing.ts` — `locales: ['tr', 'en', 'ar']`, `defaultLocale: 'tr'`
- `src/payload.config.ts` — `localization.locales`

The Languages tab in Site Settings only **toggles visibility** in the public switcher; adding a brand-new locale still requires updating both files (and adding `src/i18n/messages/<code>.json`).

## RTL
`src/app/(frontend)/[locale]/layout.tsx` sets `<html dir="rtl">` when locale is `ar`. Use Tailwind logical props (`ps-`, `pe-`, `start-`, `end-`) and add `rtl:rotate-180` on directional icons.

## Translations vs CMS content
- Static UI labels (button text, headings, sections) live in `src/i18n/messages/{tr,en,ar}.json`
- Editable marketing copy lives in Payload globals (Homepage, Site Settings, ContactInfo)
- Pattern: page reads CMS first, falls back to translation key (e.g. `homepage?.hero?.title || t('home.heroTitle')`)

## Server vs client
- Always call `setRequestLocale(locale)` before any `getTranslations()` in server components (next-intl v4 requirement)
- Wrap client subtree in `<NextIntlClientProvider messages={messages}>`
